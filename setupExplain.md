**Project Root**
- README.md: short overview of the app, tech stack, and basic run steps.
- SETUP.md: detailed install/run instructions and API notes.
- .gitignore: tells Git which generated or local files to ignore.
- .DS_Store: macOS Finder metadata, not app code.
- .git/: Git’s internal metadata folder. Files like `HEAD`, `config`, `index`, `logs`, `refs`, `objects`, and `hooks` are all part of Git history/state, not the app.

**Backend**
- backend/app.py: Flask API server and route handlers like `/timetable`, `/group`, and `/health`.
- backend/database.py: SQLite setup, seed data, and database helper functions.
- backend/requirements.txt: Python dependencies Flask needs.
- backend/timetable.db: local SQLite database file created by the backend; it stores mock/demo data.
- backend/__pycache__/: compiled Python cache files like `.pyc`; safe to delete, Python will recreate them.

**Frontend**
- frontend/index.html: the HTML entry page that mounts the React app.
- frontend/package.json: frontend project config, scripts, and npm dependencies.
- frontend/package-lock.json: exact locked npm dependency versions.
- frontend/vite.config.js: Vite setup, including dev server port and API proxy to Flask.
- frontend/dist/: production build output from `npm run build`.
- frontend/node_modules/: installed npm packages and tools. This includes third-party code like React, Vite, Babel, Rollup, and build caches; it is generated by `npm install`.

**Frontend Source**
- frontend/src/main.jsx: React entry point; renders `<App />` into the page.
- frontend/src/App.jsx: main app logic, state, login handling, and switching between personal/group views.
- frontend/src/App.css: styles for the main app shell.
- frontend/src/LoginScreen.jsx: login/user picker screen.
- frontend/src/LoginScreen.css: styles for the login screen.
- frontend/src/components/GroupView.jsx: group timetable overlay view, member toggles, and fetching group data.
- frontend/src/components/GroupView.css: styles for the group view.
- frontend/src/components/OverlayGrid.jsx: renders the color-coded group availability grid and tooltip.
- frontend/src/components/OverlayGrid.css: styles for the overlay grid.
- frontend/src/components/SlotBlock.jsx: visual block for one timetable slot in the personal grid.
- frontend/src/components/SlotBlock.css: styles for slot blocks.
- frontend/src/components/TimetableGrid.jsx: personal timetable layout that places slots by day/time.
- frontend/src/components/TimetableGrid.css: styles for the timetable grid.
- frontend/src/data/dummyData.js: mock module/user/timetable/group data used by the demo.
- frontend/src/utils/computeOverlay.js: computes the group availability overlay from member schedules.

**Generated Frontend Build Files**
- frontend/dist/index.html: generated production HTML entry.
- frontend/dist/assets/index-22ffc796.js: bundled production JavaScript.
- frontend/dist/assets/index-d71316c6.css: bundled production CSS.

**Hidden/Generated Notes**
- .DS_Store: macOS Finder metadata, same idea as the root .DS_Store.
- .bin: command shims for packages like `vite`, `rollup`, and `esbuild`.
- .vite: Vite’s cache for faster dev reloads.
- .package-lock.json: internal npm metadata inside `node_modules`.
- @babel, `react`, `vite`, `rollup`, etc.: installed third-party packages, not the app's source.

A short "what to edit for what" cheat sheet can be created to map responsibilities, for example: backend logic, UI components, styling, and generated files.

Where the real code lives
- Frontend source code (edit for UI/UX and app behavior): `frontend/src/` — React components, styles, mock data, and utilities.
- Frontend build/config: `frontend/vite.config.js`, `frontend/package.json` — edit scripts, ports, and dependency settings.
- Backend source code (edit for API/DB logic): `backend/app.py` and `backend/database.py` — these contain the Flask routes and SQLite schema/queries.
- Backend config/deps: `backend/requirements.txt` — Python packages to install in the virtualenv.

Notes:
- Ignore `frontend/node_modules/`, `frontend/dist/`, and `backend/__pycache__/` — they are generated and not for manual edits.
- The runtime database file `backend/timetable.db` contains seeded demo data; to reset it delete the file and restart the backend.
